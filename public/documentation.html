Movie API Documentation

movies_api is a Rest APi constructed in NODE.js to provide infofrmation to the user upon request to the server. The program is written in js, json, and html. The aim of this achievemnet is to learn how to use node on serverside development, and to get practive using the terminal-->

as a server side project, user experience will not be foregrounded, though the aim is of course to be as user friendly and accessible as possible. the bigger aim is to make the js files as readable and the modules as reusable as possible, so that I and other developers can understand and reuse the code.-->

  1. Return a list of ALL movies to the user
  2. Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user
  3. Return data about a genre (description) by name/title (e.g., “Thriller”)
  4. Return data about a director (bio, birth year, death year) by name
  5. Allow new users to register
  6. Allow users to update their user info (username)
  7. Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later)
  8. Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later)
  9. Allow existing users to deregister (showing only a text that a user email has been removed—more on this later)

Endpoints

Business Logic URL extension Method Request Response
get movies /movies GET - JSON Array with data for all movies
get a single movie via the title /movies/:Title GET - JSON object holding data about a single movie
Get Genre by name /movies/genres/:Name GET - JSON object holding data about the genre of a movie
Get director data by name /movies/directors/:Name GET - A JSON object holding data about the data of a director
Allow new users to register /users POST JSON object holding data about the user {"name" : "Vincent", "Password" : "XXXX", "Email": "name@email.example"} ID added via UUID JSON object containing the new user info
Allow users to update their info /users/:Username PUT - JSON object holding data about updated info
Adds movie to favorite list /users/:Username/movies/:MovieID POST - JSON object holding data about the user and the movie that was added
Delete movies from favorite list /users/:Username/movies/:MovieID DELETE - JSON object holding data about the user and the movies that was removed
Allow existing users to deregister /users/:Username DELETE - A message indicating whether the user has deregistered
